0User sends query

Natural-language biology request enters the /chat endpoint

1LLM extraction (Ollama)

Best-effort structured JSON: intent, profile fields, clarifying question, candidate queries. May fail or hallucinate — treated as suggestion only.

2Rule-based validation

Regex intent classification (priority chain). Overexpression guard. Extract organism, target, tissue, stress, multiplex. Overrides bad LLM output.

3Chitchat check

If intent = chitchat and no active experiment context → return friendly reply, no search.

not chitchat

4Clarification check

Walk the priority list for the current sub-intent. First missing required field → return question + options to user. Blocks search until answered.

all required fields filled

5Search gate

can_generate_search_queries() checks that critical fields exist for this intent. If not ready → ask the next clarification.

gate passed

6Generate candidate queries

LLM queries (filtered for concept preservation) + rule-based template queries → merge + dedup → max 5 candidates. User selects, edits, or searches all.

user confirms

7Execute search + rank

Live: concept expansion → protocols.io API. Local: TF-IDF. Profile-aware ranking with multi-signal scoring. False-positive penalties. Return top-K results + explanations.

2
Target extraction: target = "transcription factors", multiplex = true
4
modification_type = null → ASK "What kind of gene modification?"
5
Search BLOCKED — missing modification_type, tissue

"Find protocols that test for drought tolerance in Rice."

Regex: "drought" + "tolerance" → stress_tolerance_assay

Regex: "rice" → organism = rice

Search gate PASSED (organism + stress_type + readout all present)
