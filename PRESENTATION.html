3 Required vs optional handling

FieldRoleIf empty
Organism / systemhard anchorAsk clarification — blocks search
Intent · modificationhard anchorAsk clarification (editing intents)

Readout · expressiongradedProceed — used as a softer signal
Gene / constructexactProceed — exact-match filter if given
Difficulty · timeline · equipmentsoftNever blocks — tie-breaker only
Few hard anchors gate the search; everything else enriches but never blocks. Clarification fires only for missing required fields — optional gaps are fine.

4 Ranking rules — how results are ordered

Lexicographic sort key applied within the retrieved pool — each layer breaks ties of the one above.

1
Organism closeness tier
Kingdom-symmetric (plant / animal / microbe). Word-boundary + kingdom-context gated — blocks "tomato"↔"tdTomato" homonyms.

0.0
different kingdom
(plant for a mouse)
2
Title concept match
Discriminating concepts (multiplex · CRISPR · embryo · target) found in the title — strong evidence the protocol is about it, not just mentioning it. Boost only, never a filter.
3
Body relevance
Does the protocol actually do the technique (1.0) or the delivery that carries it (0.8) vs only mention it (0.0)? Separates an editing protocol from an RNA-prep that name-drops CRISPR.
4
Profile blended score
Existing weighted field signals (organism · method · readout · tissue · combined) as the final tie-breaker.
"like X" relaxation: "plants like tomato" relaxes the organism (tomato preferred → relatives → any plant); "protocols like CRISPR" relaxes the method (transformation counts, not just literal CRISPR). The other anchor stays strict.
